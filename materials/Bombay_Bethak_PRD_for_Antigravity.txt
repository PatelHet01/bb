🏪  1. Executive Summary
High-level overview of the Bombay Bethak system

Bombay Bethak is a multi-branch retail and hospitality management platform serving three business verticals — Paan Parlour, Smoke Lounge, and BB Cafe — across three locations: Gurukul (main), Bhat, and Visat. The platform is designed to be fast, mobile-first, and minimal in UI, supporting high-traffic billing workflows with exceptional UX.

🌐  4. Landing Page (Customer-Facing)
Public-facing multi-page animated experience

4.1 Overview
The landing page is a highly animated, multi-page experience designed specifically for the Bombay Bethak brand. It is NOT a single-page app. It must feature cinematic storytelling with scroll-based animations, a road trip visual narrative through the three branches, and no visible admin access points.
📝  NOTE: The landing page has NO light/dark mode. It uses a single fixed theme (dark/moody). Admin routes must never be linked or hinted at from this page.

🪑  5. My Bethak — Customer Portal
Self-service portal for customers

5.1 Overview
'My Bethak' is the customer self-service portal accessible at /my-bethak. Customers log in using only their 10-digit mobile number — no password required. This is a personal ledger and rewards hub.
5.2 Features
Tab / Section
Details
Login
Enter 10-digit mobile number. If registered, show their Bethak. If not, show registration prompt or guest view.
Ledger (Khata)
Full transaction history. Shows Levana (credit given to customer) and Deewana (payment made by customer) in customer-friendly language. Running balance displayed prominently.
Orders
History of all orders — items, quantities, amounts, dates, branch.
Rewards (GHODA)
GHODA coin balance, earned history, available rewards to redeem, betting game access.
Financial Insights
Visual charts: monthly spending, most ordered items, category-wise breakdown, average bill, loyalty score.
Profile
Name, mobile number, registration date, preferred branch.

	•	Running cart/order summary on the right / bottom (responsive)
	•	Quick Add / Remove items from cart
	•	Payment options: Cash, UPI, Credit (Levana), Advance (Deewana)

🪙  9. GHODA Rewards System
Loyalty coin system and betting game

9.1 Overview
GHODA is the internal loyalty currency of Bombay Bethak. Customers earn GHODA coins on purchases and can redeem them for rewards or use them in a betting game.
9.2 Earning Rules
	•	Configurable earn rate: e.g., 1 GHODA per ₹10 spent
	•	Bonus GHODA events can be created by admin
	•	Only registered customers (with mobile number) earn coins
	•	Coins auto-credited at billing time
9.3 Rewards CRUD (Admin)
	•	Create rewards: Name, Description, GHODA cost, Stock/availability
	•	Update and delete rewards
	•	Mark rewards as active / inactive
	•	Super admin can create global rewards; branch admin can create branch-specific rewards
9.4 Redemption
	•	Customer redeems via My Bethak portal
	•	Admin can also redeem on behalf of customer during billing
	•	Redemption deducts from GHODA balance
9.5 GHODA Betting Game
	•	A unique, simple betting game exclusive to My Bethak
	•	Customers can bet GHODA coins (not real money)
	•	Game design: simple odds-based game (e.g., coin flip, lucky draw, spin wheel)
	•	Max bet limit configurable by admin
	•	Win/loss tracked in GHODA history
	•	Cannot convert GHODA winnings to cash — internal currency only
📝  NOTE: The betting game uses GHODA coins only. It is for entertainment purposes within the app. No real money is involved. Implement responsible limits and admin controls.

💳  10. Credit System — Levana & Deewana
Customer balance and credit management

10.1 How It Works
Bombay Bethak runs a trust-based credit system for regular customers. The system tracks both credit extended to customers and advance payments made by customers.
Term
Who Owes Whom
Description
Levana
Customer owes Bombay Bethak
Customer took goods on credit. Admin records this. Customer sees 'You owe us ₹X'.
Deewana
Bombay Bethak owes Customer
Customer paid in advance. Balance is held. Customer sees 'Your advance balance: ₹X'.
10.2 Customer Identification
	•	Primary ID: 10-digit mobile number
	•	Secondary: Name (for search/display)
	•	No email, no password for customers
	•	Admin registers customers with: Name + Mobile Number
10.3 Balance Tracking
	•	Running balance shown on customer card in admin panel
	•	Full transaction log per customer
	•	Admin can manually adjust balance with reason and timestamp
	•	Balance alerts for high outstanding credit

📦  11. Inventory Management
Full inventory system with item seeding

11.1 Item Management Rules
	•	All items default to ZERO stock on creation
	•	Admin can update stock via inline edit on inventory list
	•	Items must have: Name, Category, Unit (piece/gram/ml), Price, Stock
	•	Low stock threshold is configurable per item
	•	Items can be marked inactive (hidden from billing without deletion)
11.2 Default Item List — Smoke Lounge (Gurukul + Visat)
The following is a representative item list. Actual items will be provided by client. All default to zero stock:
	•	Cigarettes (by brand — individual and pack)
	•	Gutka / Khaini / Tobacco pouches (₹1 items)
	•	Lighter, Matches
	•	Rolling papers, Filters
	•	Hookah (session-based)
	•	E-cigarettes / Vapes and refills
	•	Mouth fresheners
11.3 Default Item List — Paan Parlour (All Branches)
	•	Meetha Paan, Saada Paan, Special Paan
	•	Chocolate Paan, Fire Paan
	•	Paan ingredients (wholesale / bulk)
11.4 Default Item List — BB Cafe (Bhat Only)
	•	Hot Beverages: Chai, Coffee, Green Tea
	•	Cold Beverages: Cold Coffee, Shakes, Lemonades
	•	Snacks: Sandwiches, Maggi, Puff, Biscuits
	•	Combo Deals
📝  NOTE: This list is tentative. Admins will finalize items via the inventory CRUD. Developers should seed with these categories and zero-stock items as a baseline.
