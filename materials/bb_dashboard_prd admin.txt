SECTION 1 — Dashboard Overview & Design Philosophy
Principles that govern every screen in the admin panel
================================================================================

The Bombay Bethak admin panel is a high-traffic operational tool. At Gurukul
branch alone, hundreds of transactions happen daily. Every millisecond and every
unnecessary click costs real money. This dashboard must be the fastest, clearest,
most data-dense admin interface the team has built.

1.1 Core Design Principles
---------------------------
Principle             | Implementation
----------------------|-------------------------------------------------------------
Mobile-First          | Every dashboard screen works on a phone. Admins at Gurukul
                      | use phones behind the counter. Min tap target: 44px.
Zero Wasted Clicks    | The most common action on any screen must require at most
                      | 2 taps from landing on that screen.
Data Density          | Show more information per screen. No decorative whitespace.
                      | Every pixel earns its place.
Instant Feedback      | Every action shows result within 100ms (optimistic UI).
                      | Never let the user wonder if something worked.
Light + Dark Mode     | Both modes are fully supported. Toggle in header.
                      | Preference persisted per user.
Role-Scoped UI        | Manager, Admin, Super Admin see fundamentally different
                      | screens — not just hidden buttons.

1.2 Global Layout Structure
-----------------------------
All dashboard roles share the same shell layout. The sidebar and header are
universal. Page content changes per role and route.

Zone              | Specification
------------------|-------------------------------------------------------------
Sidebar (Desktop) | Fixed left sidebar, 240px wide. Brand logo at top. Nav links
                  | with icons. User profile + logout at bottom. Collapses to
                  | icon-only at md breakpoint.
Top Bar (Mobile)  | Sticky top bar with hamburger menu. Branch name + dark mode
                  | toggle. No horizontal scroll.
Content Area      | Scrollable main area. Max width 1280px. Padding: 24px
                  | desktop, 16px mobile.
Header            | Height 56px. Branch breadcrumb left. Dark mode toggle right.
                  | No search bar (search is page-level).
Toast             | Top-right corner. Success (green), Error (red), Warning
Notifications     | (amber). Auto-dismiss 4 seconds.

1.3 Colour System — Dashboard Only
------------------------------------
NOTE: Dashboard uses a different colour system from the landing page. Landing
page is pure black/white. Dashboard uses the admin design system below.

Token            | Hex Value | Usage
-----------------|-----------|----------------------------------------------
Brand / Primary  | #1A1A2E   | Sidebar bg, active nav, primary buttons
Accent / Orange  | #E67E22   | CTAs, highlights, active states, low-stock badges
Success          | #27AE60   | Positive balances, paid status, active items
Danger           | #C0392B   | Khata balances, void actions, zero stock
Warning          | #F39C12   | Low stock, pending status, caution
Background Light | #F8F9FA   | Page bg light mode, table alt rows
Background Dark  | #0F0F1A   | Page bg dark mode
Card Light       | #FFFFFF   | Card bg light mode
Card Dark        | #1C1C2E   | Card bg dark mode

SECTION 3 — Admin Dashboard Home Page
Branch-level command centre for daily operations
================================================================================

IMPORTANT: Manager sees a stripped-down version of this page with NO revenue,
NO profit, NO khata totals, NO analytics. Manager sees: today's order count,
low stock count, and a quick link to Billing. Nothing else.

3.1 Stat Cards Row — Admin View
---------------------------------
Four stat cards in a 2x2 grid on mobile, 4-column row on desktop. Each card is
clickable and navigates to the relevant section.

Card                     | Value Shown                                    | Click Action
-------------------------|------------------------------------------------|----------------------------
Today's Revenue          | Sum of all completed orders today              | -> Analytics, today filter
Outstanding Khata        | Total Khata balance across all customers       | -> Customers, Khata filter
(Levana)                 | in this branch                                 |
Orders Today             | Count of completed orders today                | -> Order history page
Low / Zero Stock         | Number of items at or below threshold          | -> Inventory, low-stock filter

3.2 Quick Actions Bar
-----------------------
Horizontal scrollable row of large touch-friendly quick action buttons directly
below stat cards:
▸ + New Bill — primary orange button, navigates to /admin/billing
▸ Add Customer — navigates to /admin/customers with add form open
▸ Update Stock — navigates to /admin/inventory in Bulk Update mode
▸ Record Payment — opens a quick modal to record customer Khata payment

3.3 Recent Orders Table
-------------------------
Last 10 orders for this branch. Columns: Order ID (truncated), Customer (name
or Guest), Items count, Total, Payment mode(s), Time. Each row is tappable —
expands to show itemised receipt inline.

3.4 Outstanding Khata Widget
------------------------------
A compact list of the top 5 customers with highest outstanding Khata balance for
this branch. Shows: Name, mobile, balance in red, last transaction date. Tap to
open full customer profile.

3.5 Low Stock Alerts Widget
-----------------------------
Items with stock at or below their threshold. Shows item name, current stock
(red if zero, amber if low), category. Tap opens inventory row inline edit for
quick restock.

3.6 Today's Revenue Chart
---------------------------
A simple hourly bar chart showing revenue by hour for today. Helps admin
identify peak times at a glance. Bhat branch additionally shows a BB Cafe vs
other breakdown.

3.7 Announcements Banner
--------------------------
If there is an unread announcement from Super Admin, it appears as a coloured
banner at the top of the dashboard (above stat cards). Types: Info (blue),
Warning (amber), Urgent (red). Tap to expand and mark as read.

SECTION 4 — Billing / POS Page
The highest-traffic page — built for speed above all else
================================================================================

The billing page must complete a transaction in under 10 seconds for a trained
operator. It is the most critical page in the entire system. No compromises on
speed, responsiveness, or reliability.

4.1 Three-Panel Layout
------------------------
Panel               | Specification
--------------------|-------------------------------------------------------------
Left / Top:         | Vertical tab list on desktop (140px wide). Horizontal
Category Tabs       | scrollable pills on mobile. One tab per category. Active tab:
                    | orange bg. Switching tabs is instant — all items pre-loaded.
Middle: Item Cards  | Subcategory rows with horizontal scrolling card shelves.
Grid                | Each card: item name, variant, price, qty badge. Single tap
                    | adds 1 to cart. Double tap / long press opens qty editor modal.
Right / Bottom:     | Desktop: fixed right panel 380px. Mobile: collapsible bottom
Cart                | sheet. Shows running total, item list, payment mode selector,
                    | confirm button.

4.2 Customer Bar
------------------
▸ Search input at top of page. Type name or mobile number — autocomplete
  dropdown shows matching customers.
▸ Select customer: their name, Khata balance (red), and Advance balance (green)
  shown as badges next to name.
▸ Deselect button (X) to clear customer selection.
▸ '+ New Customer' quick button beside search — opens inline mini-form: Name,
  Mobile, DOB — saves and auto-selects.
▸ If no customer selected: Khata and Advance payment modes are completely hidden
  from payment selector.

4.3 Item Card Specifications
------------------------------
State              | Visual
-------------------|-------------------------------------------------------------
Default (in stock) | Dark bg (#1A1A2E), white item name, variant in muted text,
                   | green price bottom-right
In Cart (qty > 0)  | Orange border glow, orange quantity badge top-right, card
                   | slightly elevated
Out of Stock       | Grey bg, grey text, 'Out' label, not tappable
No Price Set       | Grey bg, 'No Price' label, not tappable — admin must set
                   | price first
Inactive           | Greyed out, 50% opacity, not tappable

4.4 Payment Flow
------------------
▸ Payment mode dropdown: Cash, Online (sub: UPI / Credit Card / Debit Card),
  Khata, Advance.
▸ Khata and Advance modes only visible when a registered customer is selected.
▸ Split bill: '+ Split' button adds a second payment row. Operator sets amount
  per mode. Total must equal cart total to enable Confirm.
▸ 'Exact Cash' shortcut button: one tap confirms bill as full cash payment —
  fastest possible checkout.
▸ On confirm: order saved, stock decremented, Khata/Advance ledger updated,
  GHODA coins credited, cart clears, success toast shown. All in one atomic
  operation.
▸ Receipt shown post-confirm with order ID, items, total, payment modes.
  'New Bill' button to reset.

4.5 Offline Tolerance
-----------------------
▸ If network drops mid-session: cart is preserved in localStorage — never lost.
▸ Offline indicator shown in top bar.
▸ Order is queued and synced when connection restores. User is informed.

SECTION 5 — Inventory Page
Full stock management with inline editing and bulk operations
================================================================================

5.1 Page Header & Stats Bar
-----------------------------
▸ Title: 'Inventory'
▸ Sub-stats: '{N} active items · {N} low stock · {N} zero stock · Total Value: Rs.{X}'
▸ Low stock and Zero stock counts are clickable filters — clicking instantly
  filters table.
▸ Total Value is hidden from Manager role.
▸ Buttons (top right): 'Add Item', 'Add Disposable', 'Bulk Update'

5.2 Three Main Tabs
---------------------
Tab         | Content
------------|-------------------------------------------------------------
Active      | All active, non-archived, non-disposable items. Default tab.
Archived    | Items that have been archived. Greyed style. 'Restore' and
            | 'Delete Forever' actions.
Disposables | Internal-use items (cups, bags, etc.). Not shown on billing.
            | No selling price. Cost price only.

5.3 Filter & Search Bar
-------------------------
▸ Search: matches item name, variant, subcategory, category. Works within
  active category tab filter.
▸ Category filter pills: All | Smoke | Paan | Candy & Chewing | Beverages |
  Snacks | BB Cafe (Bhat only)
▸ Low Stock toggle pill: filters to items at or below threshold
▸ Zero Stock toggle pill: filters to items with stock = 0

5.4 Inventory Table Columns
-----------------------------
# | Column          | Details
--|-----------------|-------------------------------------------------------------
1 | Sync Dot        | Green dot = item is live on billing POS. Grey dot = item
  |                 | not appearing on billing (no price, inactive, or unassigned
  |                 | subcategory). Tooltip on hover explains why.
2 | Item & Variant  | Bold item name. Variant shown as small badge below. INACTIVE
  |                 | label in red if toggled off.
3 | Category & Sub  | Category name with icon. Subcategory below in muted text. If
  |                 | subcategory missing: amber 'Assign Subcategory' button —
  |                 | clicking opens inline subcategory dropdown.
4 | Stock           | Large clickable number. Click turns into inline input. Type
  |                 | new value, Enter or click away to save. Color: green (healthy),
  |                 | amber (low), red (zero). In Bulk Update mode: all stock cells
  |                 | become inputs simultaneously.
5 | MRP (Rs.)       | Selling price. Hidden from Manager. 'No Price' in red if not set.
6 | Value (Rs.)     | Stock x Price. Admin and Super Admin only. Hidden from Manager.
7 | Status Toggle   | Active/Inactive toggle. Click to flip. Immediate save.
  |                 | Inactive = greyed card on billing.
8 | Actions         | Always visible (not hover-only): Edit (pencil), Archive/Restore.
  |                 | Delete Forever in Archived tab (Admin only).

5.5 Inline Row Edit
---------------------
▸ Click pencil icon -> entire row becomes editable inline (no modal, no separate page).
▸ Editable fields: Name, Variant, Category, Subcategory, Unit, Price (MRP),
  Cost Price (Admin only), Low Stock Threshold.
▸ Save (green checkmark) and Cancel (grey X) buttons appear inline.
▸ Save triggers immediate DB update and real-time reflection.

5.6 Add Item Form
-------------------
Shown above the table when 'Add Item' or 'Add Disposable' is clicked. Collapses
on save or cancel.

Field                | Required | Notes
---------------------|----------|----------------------------------------------
Item Name            | YES      | Text input
Variant              | NO       | e.g. Regular, Small, 250ml
Category             | YES      | Dropdown — branch-filtered
Subcategory          | YES      | Dropdown — filtered by category selection
Unit                 | YES      | Dropdown: piece, gram, ml, kg, litre, pack, session
Selling Price (Rs.)  | YES      | Number input. Item hidden from billing until set.
Cost Price (Rs.)     | NO       | Admin only. Hidden from Manager. Used for profit calc.
Opening Stock        | YES      | Defaults to 0
Low Stock Alert At   | NO       | Defaults to 5
Active               | YES      | Checkbox. Inactive items hidden from billing.

5.7 Bulk Update Mode
----------------------
▸ Activated by 'Bulk Update' button. Entire stock column becomes editable inputs.
▸ Operator tabs through quantities rapidly — designed for restocking scenarios.
▸ Floating sticky bar at bottom: 'X items changed — Save All' button.
▸ Cancel exits bulk mode without saving. Save commits all changes in a single
  batch operation.

SECTION 6 — Customers Page
Full customer management with Khata and Advance ledger
================================================================================

6.1 Customer List
-------------------
▸ Search: by name, username, or mobile number. Real-time filter.
▸ Each customer row shows: Avatar (initials), Name, Username, Mobile, Khata
  balance (red if >0), Advance balance (green if >0), GHODA coins, order count,
  last visit date.
▸ Self-registered customers (from landing page) show a 'Self Reg' badge.
▸ Clicking a customer opens their full profile in a right panel (desktop) or
  full-screen drawer (mobile).

6.2 Add Customer Form
-----------------------
▸ Triggered by 'New Customer' button. Inline form above the list.
▸ Fields: Full Name (required), Mobile Number 10-digit (required), Date of Birth
  (required), Username (auto-generated if blank).
▸ DOB is required — used for birthday GHODA reward automation.
▸ On save: customer appears in list immediately (optimistic update). Registration
  type set to 'admin'.

6.3 Customer Profile Panel
----------------------------
Five tabs in the customer profile panel:

Tab           | Content
--------------|-------------------------------------------------------------
Profile       | Name, username, mobile, DOB, branch, registration date,
              | registration type. Khata balance card (with Record Payment
              | button). Advance balance card (with Add Advance button).
Khata Ledger  | Full transaction history. Entry types: CREDIT (goods taken),
              | PAYMENT (customer paid), ADJUSTMENT. Running balance column.
              | Newest first. Admin can add manual entries with reason.
Advance Ledger| TOPUP (advance paid), DEDUCTION (used in purchase), REFUND.
              | Running advance balance. Newest first.
Orders        | All orders with date, items list, total, payment modes, order
              | status. Tap to expand itemised view.
GHODA History | Coin earn/spend transactions. Type, amount (+ or -), reason, date.

6.4 Record Payment Modal
--------------------------
▸ Fields: Amount (large input), Payment Mode (Cash / Online), Note / Reference
  (optional)
▸ Updates Khata ledger with PAYMENT entry. Running balance reduced immediately.
▸ Supports partial payment — customer with Rs.4000 Khata can pay Rs.2000;
  remaining stays as Khata.

6.5 Add Advance Modal
-----------------------
▸ Fields: Amount (large input), Payment Mode (Cash / Online), Note
▸ Creates TOPUP entry in Advance ledger. Advance balance increases.
▸ Advance balance then appears on billing form when this customer is selected.

6.6 Devaa / Levana Terminology in UI
--------------------------------------
Context        | Admin View Label                    | Customer View (My Bethak)
---------------|-------------------------------------|---------------------------
Khata Balance  | Levana: Rs.X (collect from customer)| Your Devaa: Rs.X (you owe this)
Advance Balance| Advance: Rs.X (owed to customer)    | Your Advance: Rs.X (pre-paid balance)

SECTION 8 — Rewards (GHODA) Management
Admin control over the loyalty coin system
================================================================================

8.1 Rewards CRUD
------------------

8.3 GHODA Analytics
---------------------
▸ Total coins issued this month, total redeemed, total in circulation.
▸ Top earners list (Bethak usernames, no real names in this view).
▸ Redemption breakdown by reward type.

SECTION 9 — Analytics Page (Branch Admin View)
Deep branch-level analytics for admin decision making
================================================================================

IMPORTANT: This entire page and all its data is completely hidden from Manager
role. Route /admin/analytics returns 403 for managers. Navigation item not

9.1 Date Range Filter
-----------------------
▸ Default: Today. Options: Today, Yesterday, This Week, This Month, Last Month,

9.2 Revenue Summary Cards
---------------------------
Metric               | Details
---------------------|-------------------------------------------------------------
Total Revenue        | Sum of all completed order totals in date range
Total Orders         | Count of completed orders
Average Order Value  | Total Revenue / Total Orders
Total Cost (COGS)    | Sum of (qty x cost_price) for all sold items — Admin only
Gross Profit         | Revenue - COGS — Admin only
Total Expenses       | Sum of expense entries in date range — Admin only
Net Profit           | Gross Profit - Total Expenses — Admin only

-----------------------------

▸ X-axis: dates. Y-axis: Rs. revenue.

-----------------------
▸ Grid: 7 rows (Mon-Sun) x 24 columns (0-23 hours).
▸ Cell colour intensity proportional to order count in that hour/day combination.
▸ Colour scale: white (0 orders) to deep orange (max orders).
▸ Tap/hover: shows exact order count and revenue for that cell.
▸ Helps admin plan staffing and stock for peak hours.

9.5 Top Items
---------------

▸ Category filter: all / Smoke / Paan / Beverages / Snacks / BB Cafe.

9.6 Category Breakdown
------------------------
▸ Pie/donut chart: revenue split by category.
▸ Table below chart: each category with revenue, order count, % of total.

9.7 Customer Analytics
------------------------
▸ New customers in period (registered by admin or self-registered).
▸ Returning customers (placed order in this period AND in a previous period).
▸ Top 10 customers by spend in period: name, mobile, total spend, order count.

9.8 Export
------------
▸ Export button top-right: CSV or PDF.
▸ PDF: formatted report with date range, all charts as images, all tables.
▸ CSV: raw data rows for spreadsheet analysis.
▸ Export log: who exported what and when — visible to Super Admin only.

SECTION 11 — Super Admin System Management
Branch control, credentials, announcements, and settings
================================================================================

11.1 Branch Management
------------------------
▸ List of all branches: ID, Name, Location, Admin username, Created date, Status.
▸ Add New Branch: Name, Location, assign Admin (creates or assigns existing
  admin account).
▸ Edit branch: name and location only. Admin assignment via separate credential
  management.
▸ Architecture is extensible — adding a new branch requires no code changes,
  only data entry here.

11.2 Credential Management
----------------------------
▸ List of all admin and manager accounts: Username, Role, Branch, Last login,
  Status.
▸ Super Admin can: reset password for any admin/manager, change username,
  deactivate/reactivate account.
▸ Only Super Admin can change admin credentials. Admins cannot change their own
  username — only Super Admin can.
▸ Password reset: generates a temporary password shown once to Super Admin.
  Admin must change on next login.
▸ All credential changes are logged with timestamp and Super Admin identity.

11.3 Manager Creation (per Branch)
------------------------------------
▸ Super Admin can create managers for any branch. Branch Admin can create
  managers for their own branch.
▸ Create Manager form: Username, Password, Branch (dropdown). Manager is
  immediately active.
▸ Deactivate manager: account disabled, they cannot log in. Data preserved.
▸ Super Admin sees all managers across all branches in one table.

11.4 Announcement Board
-------------------------
▸ Super Admin sends announcements to: All Branches, or specific branch(es).
▸ Types: Info (blue banner), Warning (amber banner), Urgent (red banner —
  dismisses only after reading).
▸ Expiry date optional. After expiry, announcement is archived, not shown.
▸ Read receipts: Super Admin can see who has viewed each announcement
  (username + timestamp).
▸ Admin can send branch-level announcements to their managers. Same type system.

11.5 Export Logs
------------------
▸ Table: who exported, what (report type), date range, format (CSV/PDF),
  timestamp.
▸ Only Super Admin can see export logs.

11.6 System Settings
----------------------
▸ GHODA earn rate: Rs.X = Y coins (configurable per branch or global).
▸ GHODA birthday bonus amount.
▸ Streak milestone coin amounts (3-day, 7-day, 30-day).
▸ Referral bonus amount (referrer + referee).

  multiplier, branch scope.

-----------------

SECTION 14 — Expense Tracker
Daily operational expenses feeding into profit calculation
================================================================================

14.1 Add Expense
------------------
▸ Fields: Category (dropdown: Rent, Utilities, Supplies, Salary Advance,
  Miscellaneous, Other), Amount (Rs.), Description (text), Date (date picker,
  defaults to today).
▸ Admin logs expenses for their own branch. Super Admin can log for any branch.

14.2 Expense List
-------------------
▸ Table: Date, Category, Description, Amount, Logged By.
▸ Filter by: date range, category, branch (Super Admin only).
▸ Monthly expense total shown at top.
▸ Export: CSV / PDF.

14.3 Profit Calculation Integration
-------------------------------------
▸ Expense total feeds into: Net Profit = Gross Profit (Revenue - COGS) -
  Total Expenses.
▸ This calculation appears on Analytics page (Admin) and Super Admin analytics.
▸ Cost price per item must be set in inventory for COGS to be accurate.
